
# 个人数字名片页

## 作品简介

这是一个基于 HTML 和 CSS 的简单个人数字名片网页。它清晰地展示了我的个人信息、技能和兴趣，并提供了一个外部链接。

## 页面主要内容说明

- **个人信息**: 包括姓名、学号、专业和一句个性签名。
- **自我介绍**: 一段超过100字的个人陈述。
- **标签**: 分别展示了我的兴趣和技能。
- **联系方式**: 提供了联系方式。
- **外部链接**: 指向我的 GitHub 主页。
- **个人形象**: 包含一张个人照片。

## 本作业中我认为最能体现软件作品的一点

我认为最能体现“软件作品”而非“单个页面”的一点是 结构化的项目组织和版本控制的实践。

在这个项目中，我将 HTML、CSS 和图片资源分离到不同的文件和目录中，使得代码更易于阅读、修改和扩展。

此外，将这个项目托管在 GitHub 上，并使用 `README.md` 进行详细说明,这种规范化的文件结构、清晰的文档说明以及通过版本控制系统进行管理的方式，共同赋予了这个项目“软件作品”的属性，而不仅仅是一个孤立的网页文件。

### AI主要帮助了哪些部分
AI 主要帮助我快速生成了 `index.html` 和 `style.css` 的基础代码框架。

### 您对AI结果做了哪些修改
修改了部分css样式，对原来歪的头像图片进行了顺时针旋转31度处理，使其摆正。

### 您认为AI最不可靠的一点是什么
ai有时提供解决方案并不是最新最可靠的方案，有可能会用旧数据糊弄我，比如这次我向ai学习如何在gitee使用生成网址的功能，ai告诉我要在服务中找到Gitee Pages的选项，结果我找半天没找到，再去问ai问了好几次还是用原来的说法糊弄我，直到我试探的问ai是不是服务不提供了，结果ai才告诉我gitee已经不提供pages的服务了，然后告诉我github的pages可以用。
